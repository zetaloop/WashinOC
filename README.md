# WashinOC

基于 ESP32 (`esp-hal`) 和 Rust 的迷你洗衣机控制器固件。<br>实现了状态机驱动的洗涤程序、触摸按键控制、TM1637 显示以及深度休眠管理。

## 硬件配置

- **主控**: ESP32 (NODEMCU-32, WROOM-32E)
- **供电**: 9V (电机), 3.3V (控制逻辑)
- **输入**: 电容触摸传感器 (高电平触发)
- **显示**: TM1637 4位数码管
- **驱动**: XY-MC10 直流电机模块 (双 PWM 输入正反转)

### 引脚映射

| 组件           | 引脚 | ESP32 GPIO | 备注                                   |
| :------------- | :--- | :--------- | :------------------------------------- |
| **触摸传感器** | SIG  | GPIO 4     | RTC 域，用于 `ext0` 唤醒。高电平触发。 |
| **TM1637**     | CLK  | GPIO 16    | 推挽输出。                             |
|                | DIO  | GPIO 17    | 开漏输出 + 内部上拉 (ACK 协议需要)。   |
| **电机驱动**   | IN1  | GPIO 18    | LEDC PWM 通道 0 (正转)。               |
|                | IN2  | GPIO 19    | LEDC PWM 通道 1 (反转)。               |

## 洗涤程序

支持 4 种洗涤模式：

1. ` 5:Lo`: 5 分钟，弱 (65% 占空比，6V)
2. ` 5:Hi`: 5 分钟，强 (100% 占空比，9V)
3. `10:Lo`: 10 分钟，弱
4. `10:Hi`: 10 分钟，强

### 电机时序
运行期间，电机按以下周期循环：
`正转 (10s) -> 刹车 (1.5s) -> 反转 (10s) -> 刹车 (1.5s)`

## 交互流程

系统通过轮询状态机运行。

- **唤醒**: 设备默认处于深度休眠。触摸传感器触发 `ext0` 唤醒，进入模式选择。
- **选择**: 短按切换 4 种模式，显示屏显示当前模式。
- **启动**: 长按 (≥800ms) 启动洗涤程序。
- **运行**: 显示屏倒计时 `分:秒`，电机按时序运转。启动后 1 秒内短按无效防误触。
- **暂停**: 运行中短按暂停，电机停止，定时器冻结，显示屏闪烁。再次短按恢复。
- **取消**: 运行或暂停时，长按直接取消当前程序。
- **休眠**: 满足以下条件时，设备显示关机动画 `----` 并进入深度休眠：
  1. 洗涤程序结束。
  2. 手动取消程序。
  3. 唤醒后 10 秒内无任何操作。

## 编译与烧录

本项目使用 `no_std` Rust 编写。需要配置 Xtensa 目标工具链。

### 环境准备

1. 使用 [espup](https://github.com/esp-rs/espup) 安装 Xtensa 工具链：
   ```bash
   cargo install espup
   espup install
   ```
2. 安装 [espflash](https://github.com/esp-rs/espflash) 烧录工具：
   ```bash
   cargo install espflash
   ```

### 烧录

通过 USB 连接 ESP32。如果需要，指定串口（如 `COM3` 或 `/dev/ttyUSB0`）。

编译并烧录：

```bash
cargo build --release
espflash flash target/xtensa-esp32-none-elf/release/washin_oc
```
